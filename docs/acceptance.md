# 受け入れ基準（完成の定義）

本書は「この操作がすべて通れば β 版完成」と言えるチェックリストを定義する。  
各項目は **手動確認 or 自動テスト** のどちらかで検証する。

> **β 版の範囲**: 約 20 名の生徒 + 数名のスタッフが日常的に利用できる状態。  
> 高可用性・大規模負荷対策は対象外とし、最低限の安定運用を目指す。

---

## 1. 生徒フロー

| # | 操作 | 期待結果 | 検証方法 |
|---|------|----------|----------|
| S-01 | Google アカウントでログイン | 許可リストに `active` で登録済みのメールならログイン成功。`app_user` が作成される | 手動 + `/api/sync-user` テスト |
| S-02 | 許可されていないメールでログイン | 「許可されていないメールアドレスです」と表示。ログインできない | 手動 |
| S-03 | `pending` 状態のメールでログイン | 「利用開始準備中です」と表示。チャット画面に入れない | 手動 |
| S-04 | `revoked` 状態のメールでログイン | 「アカウントが停止されています」と表示 | 手動 |
| S-05 | テキストで質問を送信 | AI が Markdown 形式で回答し、ストリーミングで表示される | 手動 + API テスト |
| S-06 | 数式を含む質問を送信 | AI の回答中の `$...$` が KaTeX で正しくレンダリングされる | 手動 |
| S-07 | 画像を添付して質問を送信 | 画像がアップロードされ、AI が画像の内容を踏まえて回答する | 手動 |
| S-08 | 会話履歴一覧を表示 | サイドバーに過去の会話がタイトル・日付付きで一覧表示される | 手動 |
| S-09 | 過去の会話を選択 | 選択した会話のメッセージが時系列で正しく表示される | 手動 |
| S-10 | 新規会話を開始 | 「＋」ボタンで新しい空の会話が開始される | 手動 |
| S-11 | 他の生徒の会話にアクセス | RLS により取得できない（空の結果 or 403） | RLS テスト |

## 2. スタッフフロー

| # | 操作 | 期待結果 | 検証方法 |
|---|------|----------|----------|
| T-01 | Google アカウントでログイン（`role=staff`） | 管理画面にアクセスできる | 手動 |
| T-02 | 許可メール一覧を表示 | 全件表示。ステータス・ラベルで検索/絞り込み可能 | 手動 |
| T-03 | 許可メールを新規登録 | メールアドレスが `active` で追加される。監査ログが記録される | 手動 + API テスト |
| T-04 | 許可メールのステータスを変更 | `active` ↔ `pending` ↔ `revoked` の切り替えが即時反映される | 手動 |
| T-05 | CSV で一括登録 | CSV ファイルから複数メールが一括登録される | 手動 |
| T-06 | 全生徒の会話を検索 | 期間・生徒メールで絞り込み、会話一覧が表示される | 手動 |
| T-07 | 会話詳細を閲覧 | メッセージ全文・添付画像・タイムスタンプが表示される | 手動 |
| T-08 | 月次レポートを受信 | 月末に CSV + HTML メールが届く | 手動（dry-run 検証） |
| T-09 | 月次レポートを手動再実行 | 管理画面から対象月を指定して再送信できる | 手動 |

## 3. 非機能要件

| # | 項目 | 基準 | 検証方法 |
|---|------|------|----------|
| N-01 | ビルド成功 | `pnpm build` がエラーなく完了する | CI |
| N-02 | 型チェック | `pnpm typecheck` が通る | CI |
| N-03 | Lint | `pnpm lint` が通る | CI |
| N-04 | テスト | `pnpm test` が全件パス | CI |
| N-05 | 応答速度 | テキストのみの質問で、最初のストリーム到達まで p95 3 秒以内（平常時） | 手動計測 |
| N-06 | LLM フォールバック | プライマリ失敗時にフォールバックモデルで応答できる | 自動テスト |
| N-07 | セキュリティヘッダ | CSP / HSTS / X-Frame-Options が応答ヘッダに含まれる | 手動（DevTools） |
| N-08 | RLS 検証 | 学生=自分のみ、スタッフ=全件がテストで証明されている | 自動テスト |
| N-09 | XSS 防御 | `<script>` タグが Markdown 表示で除去される | 自動テスト |
| N-10 | レスポンシブ | スマートフォン（375px 幅）でチャット画面が操作可能 | 手動 |
| N-11 | requestId | すべての API レスポンスに `requestId` が含まれる | 自動テスト |

## 4. β 版で対象外とするもの（将来対応）

以下は β 版のスコープ外とし、安定運用後に検討する。

- E2E テスト（Playwright）の完全網羅
- 多言語対応
- 保護者向け共有機能
- SLA 保証（99.9% 等）
- 大規模負荷テスト（100 名超）
- データの自動物理削除 Cron
