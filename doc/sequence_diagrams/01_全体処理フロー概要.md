# 1. 全体処理フロー概要

```mermaid
sequenceDiagram
    actor Student as 生徒
    actor Staff as スタッフ
    participant FE as フロントエンド<br/>(Next.js)
    participant API as APIルート<br/>(Route Handler)
    participant Auth as Supabase Auth
    participant DB as Supabase<br/>PostgreSQL
    participant Storage as Supabase<br/>Storage
    participant AI as OpenAI API

    Note over Student, AI: === 認証フロー ===
    Student->>FE: ログイン
    FE->>Auth: signInWithPassword
    Auth-->>FE: JWT トークン

    Note over Student, AI: === チャットフロー ===
    Student->>FE: メッセージ送信
    FE->>API: POST /api/chat (Bearer)
    API->>Auth: トークン検証
    API->>AI: streamText (GPT-4o-mini)
    AI-->>API: ストリーミング応答
    API-->>FE: Streaming Response
    API->>DB: 会話・メッセージ保存

    Note over Staff, AI: === 管理フロー ===
    Staff->>FE: 管理画面アクセス
    FE->>API: GET /api/admin/* (Bearer)
    API->>Auth: トークン検証 + ロール確認
    API->>DB: Service Role でクエリ
    DB-->>API: 結果
    API-->>FE: JSON レスポンス
```
