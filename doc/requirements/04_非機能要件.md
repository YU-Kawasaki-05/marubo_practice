# 4. 非機能要件

#### NFR-01: 性能要件

| 要件ID | 要件 | 実装状況 |
|---|---|---|
| NFR-01-1 | AIチャット応答はストリーミング形式で逐次表示する | 実装済み（Vercel AI SDK `streamText`） |
| NFR-01-2 | ヘルスチェックはEdge Runtimeで低レイテンシ応答する | 実装済み |
| NFR-01-3 | 会話一覧はカーソルベースのページネーションで効率的に取得する | 実装済み（1-50件/ページ） |
| NFR-01-4 | 管理者会話検索はオフセットベースのページネーションを使用する | 実装済み（最大50件/ページ） |
| NFR-01-5 | Markdownレンダリングは`React.memo`で不要な再レンダリングを防止する | 実装済み |

#### NFR-02: セキュリティ要件

| 要件ID | 要件 | 実装状況 |
|---|---|---|
| NFR-02-1 | すべてのAPIエンドポイントはBearer トークンによる認証を必須とする | 実装済み |
| NFR-02-2 | PostgreSQL RLS（Row-Level Security）により、ユーザーは自身のデータのみアクセスできる | 実装済み |
| NFR-02-3 | 管理系APIはスタッフロール確認（`requireStaff()`）を必須とする | 実装済み |
| NFR-02-4 | CSP（Content Security Policy）ヘッダを設定する | 実装済み（`next.config.js`） |
| NFR-02-5 | Markdown出力は`rehype-sanitize`でXSS対策する | 実装済み |
| NFR-02-6 | 権限管理操作は`GRANT_ALLOWED_EMAILS`による二重認可を行う | 実装済み |
| NFR-02-7 | 機密操作（許可リスト変更、権限変更）は監査ログに記録する | 実装済み |
| NFR-02-8 | HTTPセキュリティヘッダ（HSTS, X-Frame-Options, Referrer-Policy等）を設定する | 実装済み |
| NFR-02-9 | ファイルアップロードはMIMEタイプとサイズをサーバー側で再検証する | 実装済み |
| NFR-02-10 | カメラ・マイク・位置情報のPermissions Policyを無効化する | 実装済み |

#### NFR-03: 可用性要件

| 要件ID | 要件 | 実装状況 |
|---|---|---|
| NFR-03-1 | チャット保存失敗時もAI応答のストリーミングは継続する（ログのみ出力） | 実装済み |
| NFR-03-2 | ヘルスチェックエンドポイントを提供する | 実装済み |
| NFR-03-3 | エラー時は構造化されたJSONレスポンスを返す（requestId付き） | 実装済み |
