# 13. パフォーマンス評価

### 実装されているパフォーマンス最適化

| 最適化 | 詳細 |
|---|---|
| ストリーミング応答 | `streamText` によるAI応答の逐次配信（体感待ち時間の短縮） |
| メモ化 | `MemoizedMarkdown` コンポーネントによるMarkdownレンダリングの最適化 |
| カーソルベースページネーション | 会話一覧の効率的なページング |
| データベースインデックス | 主要クエリパターンに合わせたインデックス設定 |
| Edge Runtime | ヘルスチェックはEdge Runtimeで高速応答 |
| 画像遅延読み込み | `loading="lazy"` による画像の遅延読み込み |
| CSPヘッダキャッシュ | `next.config.js` でのヘッダ設定による静的キャッシュ |

### パフォーマンス上の課題と推奨改善策

| 課題 | 詳細 | 推奨対策 |
|---|---|---|
| 会話詳細の添付取得 | 会話詳細表示時に全メッセージIDで `IN` クエリを発行している | バッチサイズの制限またはJOINクエリへの最適化 |
| 管理者会話検索 | キーワード検索がテキストの部分一致の場合、大量データでのパフォーマンス低下の可能性 | 全文検索インデックス（PostgreSQL `tsvector`）の導入 |
| CSVインポート | 大量レコードの一括処理時のメモリ使用量 | バッチ処理（チャンク分割）の実装 |
| 画像署名URL | メッセージ表示ごとに署名URLを生成している可能性 | 署名URLのクライアントサイドキャッシュ |
