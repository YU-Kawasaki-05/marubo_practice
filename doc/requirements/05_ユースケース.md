# 5. ユースケース

### UC-01: 生徒がAIに質問する

| 項目 | 内容 |
|---|---|
| ユースケースID | UC-01 |
| アクター | 生徒（studentロール） |
| 事前条件 | ログイン済み、AllowlistでActive状態 |
| 事後条件 | 会話とメッセージがDBに永続化される |

**基本フロー:**

1. 生徒がチャット画面（`/chat`）にアクセスする
2. AllowlistGuardがAllowlistステータスを確認する
3. 生徒がテキスト入力欄にメッセージを入力する
4. （任意）画像をドラッグ＆ドロップまたはファイルピッカーで添付する
5. 送信ボタンを押下する
6. フロントエンドがBearerトークン付きで `POST /api/chat` を呼び出す
7. サーバーが認証を検証し、OpenAI API にリクエストをストリーミング送信する
8. AI応答がストリーミングで画面に表示される
9. ストリーミング完了後、`onFinish` コールバックで会話・メッセージ・添付がDBに保存される
10. レスポンスヘッダ `x-conversation-id` でフロントに会話IDが返却される

**代替フロー:**

- **E1**: 未認証の場合 → 401 Unauthorized を返却
- **E2**: Allowlistに存在しない場合 → AllowlistGuardが利用不可メッセージを表示
- **E3**: OpenAI APIエラーの場合 → 500エラーをJSON形式で返却
- **E4**: DB保存失敗の場合 → ログ出力のみ（AI応答は正常に返却）

### UC-02: 生徒が会話履歴を閲覧する

| 項目 | 内容 |
|---|---|
| ユースケースID | UC-02 |
| アクター | 生徒（studentロール） |
| 事前条件 | ログイン済み、過去の会話が存在する |
| 事後条件 | なし（読み取りのみ） |

**基本フロー:**

1. チャット画面のサイドバーに会話一覧が表示される（`GET /api/conversations`）
2. 生徒が会話をクリックする
3. `GET /api/conversations/[id]` で会話詳細（メッセージ＋添付）を取得する
4. メッセージがMarkdown/LaTeXレンダリングされて表示される
5. 添付画像は署名付きURLで表示される

**代替フロー:**

- **E1**: 会話が存在しない場合 → 404 Not Found
- **E2**: 他ユーザーの会話にアクセスした場合 → RLSにより404

### UC-03: スタッフが許可リストを管理する

| 項目 | 内容 |
|---|---|
| ユースケースID | UC-03 |
| アクター | スタッフ（staffロール） |
| 事前条件 | ログイン済み、staffロールが付与されている |
| 事後条件 | 許可リストが更新され、監査ログが記録される |

**基本フロー:**

1. スタッフが管理画面（`/admin/allowlist`）にアクセスする
2. 許可リスト一覧を確認する（ステータスフィルタ、メール検索可能）
3. 新しいメールアドレスを追加する（`POST /api/admin/allowlist`）
4. または、ステータスを更新する（`PATCH /api/admin/allowlist/[email]`）
5. 監査ログが自動記録される

**代替フロー (CSV一括インポート):**

1. スタッフがCSVファイルを用意する（email, status, label, notes列）
2. CSVインポートフォームにCSVを貼り付けまたはアップロードする
3. プレビューで内容を確認する
4. `POST /api/admin/allowlist/import` で一括インポートする
5. insertまたはupsertモードで処理される

### UC-04: 管理者がスタッフ権限を管理する

| 項目 | 内容 |
|---|---|
| ユースケースID | UC-04 |
| アクター | システム管理者（staffロール＋GRANT_ALLOWED_EMAILS） |
| 事前条件 | ログイン済み、staffロール、`GRANT_ALLOWED_EMAILS`に含まれる |
| 事後条件 | 対象ユーザーのロールが変更され、監査ログが記録される |

**基本フロー:**

1. 管理者が権限管理画面（`/admin/grant`）にアクセスする
2. 現在のスタッフ一覧と操作履歴を確認する（`GET /api/admin/grant`）
3. メールアドレスを入力し、grantまたはrevokeを選択する
4. `POST /api/admin/grant` でリクエストを送信する
5. `app_user.role` と `auth.users.app_metadata.role` が更新される
6. `audit_grant` テーブルに操作ログが記録される

**代替フロー:**

- **E1**: `GRANT_ALLOWED_EMAILS`に含まれない場合 → 403 Forbidden
- **E2**: 自己剥奪を試みた場合 → エラー応答（self-revoke禁止）
- **E3**: 対象ユーザーが存在しない場合 → エラー応答

### UC-05: スタッフが生徒の会話を検索する

| 項目 | 内容 |
|---|---|
| ユースケースID | UC-05 |
| アクター | スタッフ（staffロール） |
| 事前条件 | ログイン済み、staffロール |
| 事後条件 | なし（読み取りのみ） |

**基本フロー:**

1. スタッフが会話検索画面（`/admin/conversations`）にアクセスする
2. 検索条件を入力する（メール、日付範囲、キーワード）
3. `GET /api/admin/conversations` で検索を実行する
4. 検索結果一覧が表示される（生徒名、タイトル、日時、メッセージ数）
5. 会話をクリックして詳細を展開する
6. `GET /api/admin/conversations/[id]` で全メッセージ＋添付を取得する
