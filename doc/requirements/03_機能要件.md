# 3. 機能要件

#### FR-01: ユーザー認証機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-01 |
| 機能名 | ユーザー認証 |
| 優先度 | 必須（実装済み） |
| 説明 | メールアドレスとパスワードによるユーザー認証を提供する |

- **FR-01-1**: メールアドレスとパスワードによるサインアップ（`supabase.auth.signUp`）
- **FR-01-2**: メールアドレスとパスワードによるログイン（`supabase.auth.signInWithPassword`）
- **FR-01-3**: Bearerトークンによるセッション管理（JWT形式）
- **FR-01-4**: ログイン成功時、`/admin/allowlist` へリダイレクト
- **FR-01-5**: ユーザー同期機能（`POST /api/sync-user`）— Auth ユーザーと `app_user` テーブルの同期

#### FR-02: 許可リスト（Allowlist）管理機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-02 |
| 機能名 | 許可リスト管理 |
| 優先度 | 必須（実装済み） |
| 説明 | システム利用を許可するメールアドレスを管理する |

- **FR-02-1**: 許可メール一覧の表示（ステータスフィルタ、メール検索対応）
- **FR-02-2**: 許可メールの個別追加（email, status, label, notes）
- **FR-02-3**: 許可メールのステータス更新（active / pending / revoked）
- **FR-02-4**: CSV一括インポート（insert / upsert モード対応）
- **FR-02-5**: メールアドレスの正規化（小文字化・トリム）
- **FR-02-6**: 操作履歴の監査ログ記録（`audit_allowlist` テーブル）
- **FR-02-7**: AllowlistGuardコンポーネントによるアクセス制御
- **FR-02-8**: AccountStatusBannerによるステータス通知

#### FR-03: AIチャット機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-03 |
| 機能名 | AIチャット |
| 優先度 | 必須（実装済み） |
| 説明 | OpenAI GPT-4o-miniを利用したAI質疑応答チャット |

- **FR-03-1**: テキストメッセージの送信・AI応答のストリーミング受信
- **FR-03-2**: システムプロンプトによるAIの役割設定（「親切で分かりやすい塾の先生」）
- **FR-03-3**: 数式のLaTeX形式での回答（`$...$` インライン、`$$...$$` ブロック）
- **FR-03-4**: Markdown形式での回答レンダリング（GFM、数式、サニタイズ）
- **FR-03-5**: 会話タイトルの自動生成（最初のユーザーメッセージの先頭50文字）
- **FR-03-6**: ストリーミング中のローディングインジケータ表示
- **FR-03-7**: 新規会話の作成・既存会話の選択切替

#### FR-04: 画像添付機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-04 |
| 機能名 | 画像添付 |
| 優先度 | 必須（実装済み） |
| 説明 | チャットメッセージに画像を添付してAIに質問できる |

- **FR-04-1**: 画像ファイルの選択（ファイルピッカー）
- **FR-04-2**: ドラッグ＆ドロップによる画像添付
- **FR-04-3**: 添付画像のプレビュー表示と削除
- **FR-04-4**: 署名付きアップロードURL発行（`POST /api/attachments/sign`）
- **FR-04-5**: Supabase Storageへの画像アップロード
- **FR-04-6**: メッセージ表示時の署名付き閲覧URL生成（有効期限10分）
- **FR-04-7**: 画像のライトボックス表示（拡大表示）
- **FR-04-8**: バリデーション（MIME: JPEG/PNG/WebP、サイズ: ≤5MB、枚数: ≤3枚/メッセージ）

#### FR-05: 会話履歴機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-05 |
| 機能名 | 会話履歴 |
| 優先度 | 必須（実装済み） |
| 説明 | チャット会話の永続化と閲覧機能を提供する |

- **FR-05-1**: 会話の自動保存（AI応答完了時に `onFinish` コールバックで永続化）
- **FR-05-2**: 会話一覧のサイドバー表示（カーソルベースのページネーション）
- **FR-05-3**: 会話詳細の閲覧（メッセージ＋添付画像）
- **FR-05-4**: 相対日時表示（「今日 HH:MM」「n日前」「YYYY/MM/DD」）
- **FR-05-5**: ページネーション（デフォルト20件、最大50件/ページ）

#### FR-06: 管理者向け会話検索機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-06 |
| 機能名 | 管理者会話検索 |
| 優先度 | 必須（実装済み） |
| 説明 | スタッフが全生徒の会話を検索・閲覧できる |

- **FR-06-1**: メールアドレスによる生徒検索
- **FR-06-2**: 日付範囲（from / to）による期間絞込み
- **FR-06-3**: キーワードによるメッセージ内容検索
- **FR-06-4**: ページネーション付き検索結果一覧
- **FR-06-5**: 会話詳細の展開表示（メッセージ＋添付画像）

#### FR-07: スタッフ権限管理機能

| 項目 | 内容 |
|---|---|
| 機能ID | FR-07 |
| 機能名 | スタッフ権限管理 |
| 優先度 | 必須（実装済み） |
| 説明 | 特定の管理者がスタッフ権限の付与・剥奪を行える |

- **FR-07-1**: メールアドレス指定によるスタッフ権限付与（grant）
- **FR-07-2**: スタッフ権限の剥奪（revoke）
- **FR-07-3**: 自己剥奪の防止（self-revoke 禁止）
- **FR-07-4**: `GRANT_ALLOWED_EMAILS` による操作者のホワイトリスト制御
- **FR-07-5**: 権限操作の監査ログ記録（`audit_grant` テーブル）
- **FR-07-6**: 現在のスタッフ一覧表示
- **FR-07-7**: 権限操作履歴（監査ログ）の表示
