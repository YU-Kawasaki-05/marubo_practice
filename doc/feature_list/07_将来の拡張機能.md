# 7. 将来の拡張機能

### 月次学習レポート（FR-09）

- **概要**: LLMを使用して生徒ごとの月次学習レポートを自動生成
- **現状**: `monthly_report` テーブルが作成済み（スキーマ定義完了）
- **DB設計**: status (`pending` → `generating` → `generated`/`failed`)、content (Markdown)、stats (JSON)、トークン使用量
- **環境変数**: `REPORT_LLM_MODEL`, `REPORT_LLM_API_KEY`, `REPORT_MAX_TOKENS_OUT` が用意済み
- **今後の実装**:
  - レポート生成バッチジョブ（Cron）
  - レポート閲覧UI（生徒向け・スタッフ向け）
  - メール配信連携

### メール通知（FR-10）

- **概要**: Resendサービスを利用したメール通知
- **現状**: `RESEND_API_KEY`, `MAIL_FROM` 環境変数が用意済み
- **想定用途**:
  - アカウント作成通知
  - 月次レポート配信
  - 許可リスト変更通知

### 監視・アラート（FR-11）

- **概要**: Vercel Analytics / Sentry 等による運用監視
- **現状**: `docs/operational/monitoring.md` に設計文書あり
- **想定導入**: エラートラッキング、パフォーマンスモニタリング、アラート通知

### その他の将来計画

- **OAuth連携**: Google/GitHub等のソーシャルログイン
- **マルチLLM対応**: Claude等の代替LLMへの切替
- **リアルタイム通知**: Supabase Realtime / WebSocket
- **高度な検索**: PostgreSQL全文検索（tsvector）の導入
- **APIレート制限**: ユーザーごとのリクエスト制限
