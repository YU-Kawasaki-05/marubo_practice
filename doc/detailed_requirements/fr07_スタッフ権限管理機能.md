# FR-07: スタッフ権限管理機能

## 7.1 機能概要

`GRANT_ALLOWED_EMAILS` に登録された特定の管理者のみが、ユーザーのスタッフ権限を付与（grant）・剥奪（revoke）できる。全操作は監査ログに記録される。

## 7.2 詳細仕様

### FR-07-1: 権限付与（grant）

- **エンドポイント**: `POST /api/admin/grant`
- **処理フロー**:
  1. `requireStaff()` でスタッフ認証
  2. `assertGrantAllowed()` で `GRANT_ALLOWED_EMAILS` ホワイトリスト確認
  3. `executeGrant()` でロール変更実行:
     a. 対象ユーザーの `app_user` レコードを取得
     b. `app_user.role` を `'staff'` に UPDATE
     c. `auth.users.app_metadata.role` を `'staff'` に更新（Admin API）
     d. `audit_grant` テーブルに操作ログを INSERT

### FR-07-2: 権限剥奪（revoke）

- 同じエンドポイント・フローで `action: 'revoke'` を指定
- `app_user.role` を `'student'` に変更

### FR-07-3: 自己剥奪防止

```typescript
// grant.ts 内
if (payload.action === 'revoke' && payload.email === operator.email) {
  throw new AppError(400, 'SELF_REVOKE', '自分自身の権限は剥奪できません。')
}
```

### FR-07-4: ホワイトリスト制御

```typescript
// grant.ts 内
function assertGrantAllowed(email: string): void {
  const allowedRaw = process.env.GRANT_ALLOWED_EMAILS ?? ''
  const allowed = allowedRaw.split(';').map((e) => e.trim().toLowerCase())
  if (!allowed.includes(email.toLowerCase())) {
    throw new AppError(403, 'GRANT_NOT_ALLOWED', '権限管理の操作が許可されていません。')
  }
}
```

### FR-07-6〜7: 一覧・履歴表示

- **エンドポイント**: `GET /api/admin/grant`
- **レスポンス**:
  - `staffUsers`: 現在のstaffロールユーザー一覧（`app_user` で `role = 'staff'` を検索）
  - `auditLog`: 操作履歴（`audit_grant` テーブルを新しい順に取得）

## 7.3 データ構造

```typescript
// 権限操作ペイロード
type GrantPayload = {
  email: string
  action: 'grant' | 'revoke'
}

// 操作者情報（requireStaff() の返却値）
type StaffContext = {
  authUserId: string
  appUserId: string
  email: string
}

// 権限操作結果
type GrantResult = {
  email: string
  previousRole: string
  newRole: string
  note: string
}

// 一覧レスポンス
type GrantInfoResponse = {
  staffUsers: Array<{
    id: string
    email: string
    display_name: string | null
    created_at: string
  }>
  auditLog: Array<{
    id: string
    operator_email: string
    target_email: string
    action: 'grant' | 'revoke'
    prev_role: string
    new_role: string
    created_at: string
  }>
}

// 監査ログ INSERT 型
type AuditGrantInsert = {
  request_id: string
  operator_user_id: string
  target_user_id: string
  action: 'grant' | 'revoke'
  prev_role: string
  new_role: string
}
```

## 7.4 エラーハンドリング

| エラーコード | HTTPステータス | 条件 |
|---|---|---|
| UNAUTHORIZED | 401 | 未認証 |
| FORBIDDEN | 403 | staffロールなし |
| GRANT_NOT_ALLOWED | 403 | GRANT_ALLOWED_EMAILS 外 |
| SELF_REVOKE | 400 | 自己剥奪 |
| USER_NOT_FOUND | 404 | 対象ユーザー未登録 |
