# FR-02: 許可リスト管理機能

## 2.1 機能概要

スタッフが利用を許可するメールアドレスを管理する。個別追加、ステータス更新、CSV一括インポートに対応し、全操作を監査ログに記録する。

## 2.2 詳細仕様

### FR-02-1: 許可メール一覧表示

- **エンドポイント**: `GET /api/admin/allowlist`
- **クエリパラメータ**:
  - `status`: `active` | `pending` | `revoked`（任意）
  - `search`: メール部分一致検索（任意）
- **処理**: Service Role で `allowed_email` テーブルを検索
- **レスポンス**: `AllowedEmailRow[]`

### FR-02-2: 許可メール個別追加

- **エンドポイント**: `POST /api/admin/allowlist`
- **リクエストボディ**:
  ```json
  {
    "email": "student@example.com",
    "status": "active",
    "label": "山田太郎",
    "notes": "高校2年"
  }
  ```
- **処理**:
  1. メールアドレスの正規化（小文字化・トリム）
  2. バリデーション（メール形式チェック）
  3. `allowed_email` テーブルへ INSERT
  4. `audit_allowlist` テーブルへ監査ログ INSERT（operation: `insert`）
- **レスポンス**: 201 Created + 作成レコード

### FR-02-3: 許可メールステータス更新

- **エンドポイント**: `PATCH /api/admin/allowlist/[email]`
- **リクエストボディ**: `{ status?, label?, notes? }`
- **処理**:
  1. メールアドレスのURLデコード・正規化
  2. 既存レコードの取得（prev状態の記録）
  3. `allowed_email` テーブルの UPDATE
  4. `audit_allowlist` テーブルへ監査ログ INSERT（operation: `update`、prev/next記録）

### FR-02-4: CSV一括インポート

- **エンドポイント**: `POST /api/admin/allowlist/import`
- **入力形式**:
  - JSON: `{ "csv": "email,status,label\n...", "mode": "insert" }`
  - Raw CSV: `Content-Type: text/csv`
- **CSVカラム**: `email`（必須）, `status`, `label`, `notes`
- **モード**:
  - `insert`: 新規のみ追加（既存はスキップ）
  - `upsert`: 新規追加 + 既存は上書き
- **処理**:
  1. CSV文字列のパース（RFC 4180準拠）
  2. 各行のバリデーション・正規化
  3. モードに応じた INSERT / UPSERT
  4. 各レコードの監査ログ記録（operation: `csv-import`）
- **レスポンス**: `{ inserted: number, updated: number }`
- **制約**: 1リクエスト500行以内を推奨

### FR-02-5: メールアドレス正規化

```typescript
// allowlist.ts 内の正規化ロジック
function normalizeEmail(raw: string): string {
  return raw.toLowerCase().trim()
}
```

### FR-02-7: AllowlistGuard コンポーネント

- **配置**: `/chat` ページのラッパー
- **処理**:
  1. ユーザーのAllowlistステータスを確認
  2. `active` 以外の場合、アクセス不可メッセージを表示
  3. `active` の場合、子コンポーネント（ChatInterface等）をレンダリング

## 2.3 データ構造

```typescript
// allowed_email テーブル行
type AllowedEmailRow = {
  email: string           // PK
  status: 'active' | 'pending' | 'revoked'
  label: string | null
  invited_at: string | null
  expires_at: string | null
  notes: string | null
  created_by: string | null
  updated_by: string | null
  updated_at: string
  created_at: string
}

// 作成ペイロード
type CreateAllowlistPayload = {
  email: string
  status?: 'active' | 'pending' | 'revoked'
  label?: string | null
  notes?: string | null
}

// 更新ペイロード
type UpdateAllowlistPayload = {
  status?: 'active' | 'pending' | 'revoked'
  label?: string | null
  notes?: string | null
}

// 監査ログ
type AuditAllowlistInsert = {
  request_id: string
  email: string
  prev: Record<string, unknown> | null
  next: Record<string, unknown> | null
  operation: 'insert' | 'update' | 'csv-import'
  staff_user_id: string
}
```

## 2.4 エラーハンドリング

| エラーコード | HTTPステータス | 条件 | メッセージ |
|---|---|---|---|
| UNAUTHORIZED | 401 | 未認証 | 認証が必要です |
| FORBIDDEN | 403 | staffロールなし | スタッフ権限が必要です |
| STATUS_INVALID | 400 | 不正なステータス値 | status クエリは active/pending/revoked のみ |
| DUPLICATE_EMAIL | 409 | 既存メール | メールアドレスは既に登録されています |
| CSV_PARSE_ERROR | 400 | CSV解析失敗 | CSVの解析に失敗しました |
