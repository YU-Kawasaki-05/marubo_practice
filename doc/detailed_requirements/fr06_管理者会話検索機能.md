# FR-06: 管理者会話検索機能

## 6.1 機能概要

スタッフが全生徒の会話を検索・閲覧する機能。メールアドレス、日付範囲、キーワードによる検索とページネーション付き結果表示に対応する。

## 6.2 詳細仕様

### FR-06-1〜6-3: 検索条件

- **エンドポイント**: `GET /api/admin/conversations`
- **検索パラメータ**:

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| email | string | 任意 | 生徒メールアドレス（部分一致） |
| from | string (date) | 任意 | 開始日（YYYY-MM-DD） |
| to | string (date) | 任意 | 終了日（YYYY-MM-DD） |
| keyword | string | 任意 | メッセージ内容検索 |
| page | number | 任意 | ページ番号（デフォルト: 1） |
| limit | number | 任意 | 件数（デフォルト: 20、最大: 50） |

- **処理**: `adminConversations.listConversations()` でService Role を使用してクエリ実行
- **認可**: `requireStaff()` でスタッフロール確認

### FR-06-5: 会話詳細展開

- **エンドポイント**: `GET /api/admin/conversations/[id]`
- **処理**: `adminConversations.getConversationDetail()` でメッセージ＋添付を取得
- **認可**: `requireStaff()` でスタッフロール確認

## 6.3 データ構造

```typescript
// 検索フィルター
type ConversationSearchFilters = {
  email?: string
  from?: string   // YYYY-MM-DD
  to?: string     // YYYY-MM-DD
  keyword?: string
  page: number
  limit: number
}

// 検索結果
type ConversationSearchResult = {
  results: Array<{
    id: string
    title: string
    user_email: string
    created_at: string
    message_count: number
  }>
  total: number
  page: number
  limit: number
}
```

## 6.4 エラーハンドリング

| エラーコード | HTTPステータス | 条件 |
|---|---|---|
| UNAUTHORIZED | 401 | 未認証 |
| FORBIDDEN | 403 | staffロールなし |
| NOT_FOUND | 404 | 会話IDが存在しない |
