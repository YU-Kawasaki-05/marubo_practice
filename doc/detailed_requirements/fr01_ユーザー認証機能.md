# FR-01: ユーザー認証機能

## 1.1 機能概要

Supabase Auth を利用したメールアドレス/パスワード認証を提供する。認証後、`app_user` テーブルとの同期を行い、許可リスト（Allowlist）によるアクセス制御を実施する。

## 1.2 詳細仕様

### FR-01-1: サインアップ

- **処理**: `supabase.auth.signUp({ email, password })` を呼び出す
- **画面**: `/login` ページ（Client Component）
- **フロー**:
  1. ユーザーがメールアドレスとパスワードを入力
  2. 「サインアップ」ボタンを押下
  3. Supabase Auth にサインアップリクエストを送信
  4. 成功時: `/admin/allowlist` へリダイレクト（`router.push`）
  5. 失敗時: エラーメッセージを画面に表示

### FR-01-2: ログイン

- **処理**: `supabase.auth.signInWithPassword({ email, password })` を呼び出す
- **画面**: `/login` ページ（Client Component）
- **フロー**: サインアップと同様のUIで、ログインモードを切り替え可能

### FR-01-3: JWTセッション管理

- **トークン形式**: Supabase発行のJWT
- **認証方式**: `Authorization: Bearer <token>` ヘッダ
- **検証方法**: `supabase.auth.getUser(token)` でサーバー側検証
- **有効期限**: Supabase Auth のデフォルト設定に従う
- **JWTクレーム確認**: `iss`, `aud`, `exp` をサーバー側で検証（デバッグログあり）

### FR-01-5: ユーザー同期（sync-user）

- **エンドポイント**: `POST /api/sync-user`
- **処理フロー**:
  1. Bearer トークンからユーザー情報を取得
  2. `allowed_email` テーブルでメール許可状態を確認
  3. ステータス別分岐:
     - `active`: 正常続行
     - `revoked`: 403 `ALLOWLIST_REVOKED`（アカウント停止）
     - `pending`: 409 `ALLOWLIST_PENDING`（準備中）
     - 未登録: 403 `ALLOWLIST_NOT_FOUND`
  4. `app_user` テーブルへのUpsert:
     - 既存ユーザー: メールアドレスのみ更新（ロールは保持）
     - 新規ユーザー: INSERT（ロールは `student` デフォルト）

## 1.3 データ構造

```typescript
// ログインフォーム入力
type LoginForm = {
  email: string
  password: string
}

// sync-user レスポンス
type SyncUserResponse = {
  requestId: string
  data: {
    appUserId: string  // app_user.id
    role: 'student' | 'staff'
    allowedEmailStatus: 'active' | 'pending' | 'revoked'
  }
}
```

## 1.4 エラーハンドリング

| エラーコード | HTTPステータス | 条件 | メッセージ |
|---|---|---|---|
| UNAUTHORIZED | 401 | トークン無効/期限切れ | ログインセッションが無効です。 |
| ALLOWLIST_NOT_FOUND | 403 | メール未登録 | 許可されていないメールアドレスです。 |
| ALLOWLIST_REVOKED | 403 | アカウント停止 | アカウントが停止されています。 |
| ALLOWLIST_PENDING | 409 | 準備中 | 利用開始準備中です。 |
